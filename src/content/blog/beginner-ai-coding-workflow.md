AI에게 “앱 하나 만들어줘”라고 요청하면 빠르게 코드가 나옵니다. 하지만 처음 배우는 단계에서는 완성된 코드를 한 번에 받는 것보다 작은 단위로 질문하고 확인하는 편이 훨씬 도움이 됩니다.

## 1. 목표를 한 문장으로 적기

먼저 만들고 싶은 결과를 기술 용어 없이 적어보세요.

> 사용자가 할 일을 입력하고 완료 여부를 표시할 수 있는 간단한 웹 페이지를 만든다.

이 문장이 작업 범위를 지키는 기준이 됩니다. 로그인, 데이터베이스, 알림처럼 지금 필요하지 않은 기능은 뒤로 미룹니다.

## 2. AI에게 계획부터 요청하기

바로 코드를 요청하기 전에 필요한 파일과 작업 순서를 설명해 달라고 합니다.

```text
HTML, CSS, JavaScript를 처음 배우고 있어.
할 일 목록 페이지를 만들기 전에 필요한 구조와 작업 순서를
초보자가 이해할 수 있게 설명해 줘. 아직 코드는 작성하지 마.
```

계획을 먼저 보면 모르는 개념을 발견하고 범위를 줄일 수 있습니다.

## 3. 한 번에 한 기능만 구현하기

화면 구성, 입력, 목록 추가, 완료 처리, 저장 기능을 나눠서 진행합니다. 각 단계가 동작하는지 확인한 뒤 다음 기능으로 넘어가세요.

오류가 생겼다면 전체 코드를 다시 만들어 달라고 하기보다 오류 메시지와 기대한 동작을 함께 전달하는 것이 좋습니다.

## 4. 코드의 이유를 질문하기

AI가 만든 코드에서 이해되지 않는 줄을 그대로 두지 마세요.

```text
이 함수가 어떤 순서로 실행되는지 설명해 줘.
각 줄이 필요한 이유와 제거했을 때 생기는 문제도 알려줘.
```

설명을 들은 뒤 변수 이름을 바꾸거나 작은 동작을 직접 수정해 보면 이해가 훨씬 오래 남습니다.

## 5. 검증 결과를 함께 전달하기

“안 돼”라는 말만으로는 AI도 원인을 좁히기 어렵습니다. 다음 정보를 함께 전달하세요.

- 실행한 명령 또는 클릭한 순서
- 기대한 결과
- 실제 결과
- 오류 메시지 전문
- 최근에 바꾼 부분

## AI는 운전자가 아니라 내비게이션

AI 코딩 도구는 다음 방향을 빠르게 보여주는 내비게이션에 가깝습니다. 목적지와 이동 여부를 결정하고, 잘못된 길인지 확인하는 역할은 여전히 사용자에게 있습니다.

작은 기능을 계획하고, 구현하고, 검증하고, 설명하는 흐름을 반복하면 AI의 속도를 활용하면서도 개발 실력을 함께 쌓을 수 있습니다.
