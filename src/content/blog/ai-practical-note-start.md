AI에게 질문하면 그럴듯한 답은 금방 얻을 수 있습니다. 문제는 그 답을 보고서, 기획안, 고객 안내문처럼 **틀리면 곤란한 업무 자료**에 넣을 때입니다. 링크가 있어도 실제로 주장을 뒷받침하지 않거나, 작년 정책을 최신 정보처럼 설명하는 경우가 있기 때문입니다.

이 글에서는 가상의 사례 하나를 끝까지 처리해 보겠습니다.

> 팀장이 “우리 팀 문서를 생성형 AI에 올려도 되는지, 무료 도구 기준으로 조사해 주세요”라고 요청했다.

결론부터 말하면 검색형 AI 한 번으로 끝내지 않습니다. **후보 찾기 → 범위 제한 → 원문 확인 → 판단 기록**의 네 단계를 거치면 속도를 크게 잃지 않으면서도 결과의 신뢰도를 높일 수 있습니다.

![자료 수집부터 사람의 최종 검토까지 이어지는 AI 조사 흐름](/images/posts/ai-practical-workflow.jpg)

## 준비물과 역할을 먼저 나눈다

이 사례에서는 도구를 세 개만 씁니다.

| 역할 | 도구 | 이 도구를 쓰는 이유 |
| --- | --- | --- |
| 최신 자료 후보 찾기 | Perplexity | 웹 검색 결과에 출처 링크가 붙어 있어 탐색 시작점으로 편하다 |
| 지정한 문서끼리 비교 | NotebookLM | 업로드한 출처를 바탕으로 답하고 인라인 인용 위치를 확인할 수 있다 |
| 최종 판정 | 원문 브라우저 + 메모 | 약관·도움말의 문장을 사람이 직접 확인하고 날짜를 기록한다 |

핵심은 “어떤 AI가 가장 똑똑한가”가 아닙니다. 검색은 Perplexity, 제한된 자료 비교는 NotebookLM, 최종 책임은 사람이 맡는 식으로 **역할을 분리**하는 것입니다.

## 1단계: 검색 질문을 ‘결정에 필요한 항목’으로 쪼갠다

“생성형 AI 보안 정책 알려줘”라고 물으면 너무 넓은 답이 나옵니다. 실제 결정에 필요한 항목으로 질문을 바꿉니다.

```text
개인용 무료 생성형 AI 서비스에 회사 문서를 입력하려고 한다.
다음 항목별로 각 서비스의 공식 문서만 찾아 표로 정리해 줘.

1. 입력 데이터가 모델 학습에 사용되는가
2. 사용자가 학습 사용을 끌 수 있는가
3. 입력 및 대화의 보관·삭제 정책은 무엇인가
4. 기업용 요금제와 개인용 요금제의 정책 차이는 무엇인가

각 셀에는 공식 출처 URL과 문서의 최종 확인 날짜를 붙여 줘.
확인할 수 없는 내용은 추측하지 말고 ‘확인 필요’라고 표시해 줘.
```

이 프롬프트의 중요한 부분은 세 가지입니다.

- 비교 열을 미리 지정해 결과 형식을 고정합니다.
- 블로그나 커뮤니티가 아닌 **서비스 제공사의 공식 문서**를 요구합니다.
- 모르는 내용을 채우지 않도록 `확인 필요`라는 탈출구를 줍니다.

검색 결과에서 링크를 바로 믿지 말고 후보만 수집합니다. 개인정보 처리방침, 데이터 사용 도움말, 요금제별 보안 문서처럼 성격이 다른 문서를 최소 두 종류 이상 확보하는 편이 좋습니다.

## 2단계: NotebookLM에 공식 문서만 넣어 범위를 잠근다

후보 링크를 열어 실제 공식 도메인인지 확인한 뒤 NotebookLM의 새 노트북에 추가합니다. NotebookLM은 웹 URL, PDF, Google Docs 등 여러 형식의 출처를 받을 수 있고, 답변의 인용을 누르면 해당 근거 위치로 이동할 수 있습니다.

이제 질문을 이렇게 바꿉니다.

```text
선택된 출처만 사용해서 아래 표를 작성해 줘.

열: 서비스 / 개인 무료 계정 / 기업 계정 / 사용자가 바꿀 수 있는 설정 / 근거 출처

‘학습에 사용하지 않는다’와 ‘보관하지 않는다’를 같은 의미로 취급하지 마.
출처끼리 충돌하면 하나를 선택하지 말고 두 주장을 모두 적어 줘.
정책 적용 대상이 불분명하면 정확히 어떤 계정이 불분명한지 표시해 줘.
```

여기서 `학습`과 `보관`을 구분한 이유가 중요합니다. 모델 개선에 사용하지 않는다는 말이 서버에 전혀 저장되지 않는다는 뜻은 아닙니다. 이런 용어 차이를 프롬프트에 명시하지 않으면 AI가 보기 좋은 한 문장으로 합쳐 버릴 수 있습니다.

## 3단계: 인용문이 ‘바로 그 주장’을 증명하는지 확인한다

AI가 만든 표에서 중요한 셀마다 인용을 누릅니다. 다음 네 가지를 확인합니다.

1. **주어**: 개인 계정 이야기인지, 기업용 상품 이야기인지
2. **동사**: 저장, 학습, 공유, 삭제 중 무엇을 말하는지
3. **조건**: 기본값인지, 사용자가 설정을 바꾼 뒤의 상태인지
4. **시점**: 문서가 현재 정책인지, 종료된 기능을 설명하는지

예를 들어 AI가 “데이터는 학습에 사용되지 않는다”고 요약했지만 원문은 “특정 기업용 제품의 입력은 기본적으로 모델 학습에 사용되지 않는다”고 적혀 있을 수 있습니다. 이 경우 개인 무료 계정에 같은 결론을 적용하면 안 됩니다.

검증하면서 발견한 문제는 아래처럼 짧게 기록합니다.

```text
[확인일] 2026-07-15
[주장] 무료 계정 입력은 모델 개선에 사용되지 않는다.
[판정] 근거 부족
[이유] 인용된 문서는 기업용 계정에만 적용됨.
[다음 행동] 개인 계정의 데이터 제어 도움말을 별도로 확인.
```

이 네 줄이 있으면 나중에 정책이 바뀌어도 어디부터 다시 확인해야 하는지 알 수 있습니다.

## 4단계: 보고서에는 사실·해석·권고를 분리해 쓴다

조사 메모를 바로 결론으로 복사하지 않습니다. 다음 형식으로 정리하면 읽는 사람이 판단 근거를 추적하기 쉽습니다.

```text
사실: 공식 도움말에는 개인 계정과 기업 계정의 데이터 사용 조건이 다르게 설명되어 있다.

해석: 무료 개인 계정에 회사 원문을 그대로 입력하면 조직이 요구하는 통제 수준을 확인하기 어렵다.

권고: 고객명, 계약 금액, 내부 코드 등 식별 가능한 정보는 입력하지 않는다. 업무 도입 전에는 회사가 승인한 기업용 계정과 보안 설정을 사용한다.
```

`사실`에는 출처로 확인한 내용만, `해석`에는 그 사실이 우리 상황에서 의미하는 바를, `권고`에는 실제 행동을 적습니다. 세 문장을 섞지 않으면 “AI가 그렇게 말했다”가 아니라 조직이 어떤 근거로 결정했는지가 남습니다.

## 20분 안에 끝내는 최소 검증 체크리스트

시간이 부족한 업무라면 아래 여섯 항목만이라도 확인합니다.

- 질문에 대상 계정, 지역, 날짜를 적었는가
- 공식 문서 링크가 최소 두 개 이상 있는가
- 링크를 직접 열어 AI 요약과 원문을 대조했는가
- `학습`, `저장`, `공유`, `삭제`를 구분했는가
- 확인하지 못한 셀을 공란이나 `확인 필요`로 남겼는가
- 최종 문서에 출처와 확인 날짜를 기록했는가

이 흐름은 보안 정책뿐 아니라 가격 비교, 법규 변경 조사, 소프트웨어 기능 비교에도 그대로 적용할 수 있습니다. AI는 조사 시간을 줄여 주지만, **근거의 적용 범위를 판단하는 일**까지 대신해 주지는 않습니다.
